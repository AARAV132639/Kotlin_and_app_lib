# File Handling in Kotlin

*File Handling* : Persisting data outside the program

*Serialization* : Converting objects into a storable/transmittable format and reconstructing them later.

- File Handling is the process of creating, reading, writing, appending, updating and deleting files to persist data outside the application's memory

- The process of converting an object into a storable or transmittable format (such as JSON, XML or binary) so that it can be saved to a file, database or sent over a network. The reverse process is called *deserialization*
